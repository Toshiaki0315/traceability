# Blockchain Traceability Sample (PBFT・コンソーシアム型学習用)

このプロジェクトは、製造・サプライチェーンの工程データ記録を題材に、ブロックチェーンの基本原理と**エンタープライズ向け（コンソーシアム型）ブロックチェーン**の仕組みを学ぶためのPythonによる概念実証（PoC）サンプル実装です。

## プロジェクトの進化と特徴

初期のシンプルなハッシュチェーン構造から段階的に機能を拡張し、現在（ステップ3完了時点）は実際の業務システムに近い以下の仕組みを備えています。

### 1. 電子署名によるアイデンティティ管理（Oracle問題への対策）
データが「誰によって」記録されたかを保証するため、公開鍵暗号（RSA）を導入しています。
- 納入業者、加工工場、倉庫といった各参加者（ノード）は専用の鍵ペアを持ちます。
- データを送信する際は秘密鍵で署名し、ネットワーク全体で公開鍵を使って改ざん・なりすましを検知します。

### 2. 複数ノードによるP2Pシミュレーション（中央集権・51%攻撃への対策）
単一の管理者がチェーンを持つのではなく、複数の参加者が独立した台帳（Nodeオブジェクト）を持つ分散型ネットワークをシミュレーションしています。
- 各ノードは「Leader（リーダー）」または「Replica（レプリカ）」の役割を持ちます。
- トランザクション（取引）が発生すると、ネットワーク全体へブロードキャストされます。

### 3. トランザクションの分離とPBFT合意形成（準備段階）
データが発生して即座にブロック化されるのではなく、以下のフェーズを踏むように設計されています。
- **未承認プール（Mempool）**: 受信したトランザクションは署名検証後、一時的に `pending_transactions` に蓄積されます。
- **ブロックの提案**: リーダーノードが未承認トランザクションをまとめて新しいブロック候補を作り、ネットワーク全体に提案（PRE_PREPAREメッセージ）します。

## 製造業DXにおけるユースケース

食品、医薬品、化粧品など、厳格な品質管理基準（HACCPやGMPなど）が求められる製造現場において、本仕組みをIoTエッジデバイスと連携させることで以下のような効果が期待できます。
- **監査対応の迅速化:** 改ざん不可能な温度管理・加工ログによる強固な品質証明。
- **迅速なリコール対応:** 不良品発生時、対象ロットのピンポイントな追跡と影響範囲の特定。

## 実行環境の構築

### 前提条件
- **Python 3.7以上** がインストールされていること。

### 必要なライブラリのインストール
電子署名を利用するため、外部ライブラリ `rsa` が必要です。以下のコマンドでインストールしてください。
```bash
pip install rsa
```

## 実行方法

ターミナルを開き、以下のコマンドを実行してください。

```bash
python traceability.py
```

### 実行内容の流れ（現在のシミュレーション）
1. **ノードの初期化とP2P構築**: 「納入業者」「加工工場（Leader）」「倉庫」の3ノードが起動し、相互にピア登録を行います。
2. **トランザクション送信**: 納入業者が「原材料受け入れ」のデータを自身の秘密鍵で署名し、ネットワークにブロードキャストします。
3. **トランザクションのプール**: 他のノードがデータを受信し、署名を検証した上で未承認リスト（`pending_transactions`）に追加します。
4. **リーダーによるブロック提案**: 加工工場（Leader）が未承認のトランザクションをまとめ、新しい「ブロック候補」を作成して全ノードへ提案（PRE_PREPAREメッセージ）します。（※今後のステップ4で多数決による確定処理が追加されます）

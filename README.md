# Blockchain Traceability Sample (PoC / 学習用)

このプロジェクトは、ブロックチェーンの基本原理（ハッシュチェーンと改ざん検知）を製造・サプライチェーンの工程データ記録に当てはめた、Pythonによる概念実証（PoC）および学習用のサンプル実装です。

## プロジェクトの位置づけと注意点（本番利用に向けて）

本コードは、ブロックチェーンがどのようにデータの連続性を担保し、**「記録後のデータ改ざんを検知する」**かを理解するためのミニマムな実装です。
実際の業務システムや本番環境へ適用する際は、以下の点に留意し、設計を拡張する必要があります。

1. **中央集権型の限界と「51%攻撃」**
   本コードはシングルノードで動作するため、システム管理者がチェーン全体のハッシュを再計算してしまえば改ざんを隠蔽できます。また、実際の企業間（コンソーシアム）チェーンにおいても、一部の参加企業が結託してネットワークの過半数を支配する「51%攻撃」のリスクがあります。本番環境では、参加者の署名を必須とする合意形成（PBFTなど）の導入や、外部のパブリックチェーンへのハッシュ値のアンカリングが必要です。

2. **オラクル問題（物理とデジタルの境界）**
   本システムが保証するのは「ブロックチェーンに記録された**後**のデータ」の整合性です。入力元のセンサーが故障していたり、通信経路上でデータが書き換えられた場合、チェーンは「嘘のデータ」を真実として強固に記録してしまいます（Garbage In, Garbage Out）。実運用では、エッジデバイス側での直接的なデジタル署名付与など、物理レイヤーを含めたセキュアな設計が不可欠です。

## 製造業DXにおけるユースケース

食品、医薬品、化粧品など、厳格な品質管理基準（HACCPやGMPなど）が求められる製造現場において、本仕組みをIoTエッジデバイスと連携させることで以下のような効果が期待できます。

- **監査対応の迅速化:** 改ざん不可能な温度管理・加工ログによる強固な品質証明。
- **迅速なリコール対応:** 不良品発生時、対象ロットのピンポイントな追跡と影響範囲の特定。

## プロジェクト概要

各工程（ブロック）が前の工程のハッシュ値を保持することで、データの改ざんを困難にし、原材料の入荷から製品の出荷までの履歴を記録します。

### 主な機能
- **ハッシュ値による改ざん検知**: 各ブロックは自身のデータと前ブロックのハッシュから独自のハッシュを計算します。
- **データリンク**: 前のブロックのハッシュ（Previous Hash）を参照することで、時系列の連続性を保証します。
- **バリデーション**: チェーン全体の整合性を一括で検証し、途中のデータが書き換えられていないかチェックします。

## 実行環境の構築

### 前提条件
- **Python 3.7以上** がインストールされていること。
- 標準ライブラリ（`hashlib`, `json`, `time`）のみを使用しているため、追加のインストールは不要です。

### セットアップ
1. このリポジトリをクローンするか、ソースコードを `traceability.py` という名前で保存します。

## 実行方法

ターミナルを開き、以下のコマンドを実行してください。

```bash
python traceability.py
```

### 実行内容の流れ
1. **ジェネシスブロックの生成**: システムが初期化されます。
2. **工程の追加**: 「原材料受け入れ」「加熱処理」「出荷」の3つの工程データが順次記録されます。
3. **チェーンの表示**: 記録された全データとハッシュ値がコンソールに出力されます。
4. **正当性検証**: 正常な状態でデータが整合しているか検証されます。
5. **改ざんシミュレーション**: 工程2の温度データを強制的に書き換え、検証機能がデータの不整合（改ざん）を正しく検知する様子を確認できます。
